# ESLint Prettier Stylelint 区别

- Prettier 侧重代码风格检查，比如数组元素之间是否有空格 ( [a,b,c] or [a, b, c] )。

- ESLint 侧重代码质量检查，比如比较时用==会提示你使用===，或者 map() 中使用 if else 有个分支没有 return 时会提示。**这些质量问题可能会引起 Bug**。

- Stylelint 用于 CSS 代码检查。

<!--truncate-->

以上几个工具通过 npm 安装之后, 只能通过命令行运行来进行代码检查。

而在 VSCode（或其他编辑器）安装相应插件之后, 插件会读取配置文件（.eslintrc、.prettierrc、.stylelintrc）, 并判断格式是否有问题，这是使用一些编辑器功能的前提。

:::tip

VSCode 的保存时自动格式化、在格式有问题处下划线提示等功能就是依靠的插件。

:::

:::caution 注意

不要跳过本地安装！插件会选择本地版本的格式化工具，从而确保同一项目在任何情况下格式化后的效果是相同的。如果本地没有安装，插件将会使用最新的版本。

:::
